Alucinaciones

El fenómeno más peligroso de los LLMs: generar información falsa con total confianza.

¿Qué es una Alucinacion?

Una alucinación ocurre cuando un LLM genera texto que suena coherente y confiado pero que es factualmente falso, inventado o incorrecto. No es un error de software ni un bug: es una consecuencia directa de cómo funcionan estos modelos.

Los LLMs no acceden a una base de hechos verificados — predicen el siguiente token basándose en patrones estadísticos. Si el patrón más probable lleva a una respuesta falsa, el modelo la genera igualmente, sin saber que está mintiendo.

¿Por qué Ocurren?

Presion estadistica

El modelo siempre produce el token mas probable. Si no sabe la respuesta, igual genera algo plausible — el silencio no es una opcion en su arquitectura.

Datos de entrenamiento

Si el texto de entrenamiento contenia errores, sesgos o informacion desactualizada, el modelo los aprende. Basura entra, basura sale.

Limite de conocimiento

Los modelos tienen un "corte de conocimiento" (training cut-off). Eventos posterior a ese fecha simplemente no existen para el modelo.

Tipos de Alucinacion

Como Minimizarlas

1.
Verificar siempre el output. Nunca copies código o datos sin revisarlos. Un LLM confiado puede estar completamente equivocado.
2.
Pedir fuentes y luego buscarlas tú mismo. El modelo puede inventar referencias bibliográficas que suenan reales.
3.
Usar temperature baja (0.0–0.3) para tareas factuales. Reduce la aleatoriedad y las invenciones creativas.
4.
Usar RAG (Retrieval Augmented Generation). Proporcionar documentos reales al modelo para que base sus respuestas en ellos y no en su memoria.
5.
Decirle que admita incertidumbre. "Si no sabes la respuesta, dilo claramente." Algunos modelos responden mejor ante esta instrucción.
6.
Usar modelos con búsqueda web. ChatGPT con browsing, Perplexity o Claude con herramientas pueden verificar información en tiempo real.
