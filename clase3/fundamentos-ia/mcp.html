MCP — Protocolo de Contexto

Model Context Protocol: el estándar abierto para conectar LLMs con el mundo real.

¿Qué es MCP?

Model Context Protocol (MCP) es un protocolo abierto creado por Anthropic en noviembre de 2024. Define una forma estandarizada de conectar modelos de IA con herramientas externas, fuentes de datos y servicios — de manera similar a cómo HTTP estandarizó la comunicación web.

Analogia:
MCP es al AI lo que USB es a los periféricos.
Antes de USB, cada dispositivo tenía su propio conector.
MCP estandariza cómo los modelos se conectan con herramientas.

El modelo nunca llama directamente a GitHub — el servidor MCP actúa como intermediario seguro.

¿Qué puede proveer un servidor MCP?

Tools (Herramientas)

Funciones que el modelo puede llamar con parámetros. Ejemplo: create_file(path, content)

  • Crear/editar archivos
  • Ejecutar comandos
  • Llamar APIs

Resources (Recursos)

Datos que el servidor expone para que el modelo los lea. Ejemplo: el contenido de un archivo o una base de datos.

  • Archivos del proyecto
  • Registros de base de datos
  • Logs y telemetría

Prompts (Plantillas)

Plantillas de prompts predefinidas que el servidor puede instanciar. Estandariza cómo el modelo interactúa con el servicio.

  • Instrucciones de sistema
  • Plantillas de tarea
  • Contexto de dominio

Servidores MCP Populares

@modelcontextprotocol/server-github

Gestión de repos, PRs, issues desde el modelo.

@modelcontextprotocol/server-filesystem

Leer y escribir archivos locales de forma segura.

@modelcontextprotocol/server-postgres

Consultar y modificar bases de datos PostgreSQL.

@modelcontextprotocol/server-brave-search

Búsqueda web en tiempo real a través de Brave.
