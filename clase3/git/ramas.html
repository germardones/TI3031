Ramas y Branching

Las ramas permiten trabajar en paralelo sin tocar el código estable. Pasa el cursor sobre cada commit.

¿Por qué usar ramas?

Con Ramas

  • • El código en main siempre funciona
  • • Experimenta sin miedo
  • • Trabaja en múltiples features
  • • Revisión antes de fusionar

Sin Ramas

  • • Código roto en producción
  • • No puedes probar ideas
  • • Conflictos constantes en equipo
  • • Todo o nada al subir cambios

Flujo Típico

  • 1. Crea rama feature/x
  • 2. Trabaja y haz commits
  • 3. Abre Pull Request
  • 4. Revisión → Merge a main

Comandos de Ramas

Crear y Navegar
$ git branch # listar ramas
$ git branch feature/login # crear
$ git checkout feature/login # cambiar
$ git checkout -b feature/x # crear+cambiar
Fusionar y Eliminar
$ git checkout main # volver a main
$ git merge feature/login # fusionar
$ git branch -d feature/login # eliminar
$ git branch -a # ver todas (locales+remotas)
