El Flujo Completo en Comandos

terminal — flujo completo
# 1. Ver qué archivos were modificados
$git status
# 2. Agregar archivos al Staging Area
$git add index.html    # un archivo
$git add .           # todos los cambios
# 3. Crear el commit (foto del estado)
$git commit -m "feat: agrega sección hero"
# 4. Subir al servidor remoto
$git push origin main
# 5. Descargar cambios del servidor
$git pull origin main

Regla de Oro del Commit

Un commit debe representar un solo cambio lógico y completo. No hagas commits con 30 cambios mezclados, ni uno por cada línea editada. La pregunta clave es: "¿Puedo describir este cambio en una sola oración clara?" Si sí, es momento de hacer commit.
