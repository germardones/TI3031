Código HTML — Línea a Línea

Anatomía de un Formulario

formulario.html 

<form
  action="/enviar"
  method="POST"
  novalidate>

  
  <div class="campo">

    
    <label for="nombre">Nombre completo</label>

    
    <input
      type="text"
      id="nombre"
      name="nombre"
      placeholder="Juan Pérez"
      required
      minlength="2">
  </div>

  
  <div class="campo">
    <label for="email">Correo electrónico</label>
    <input
      type="email"
      id="email"
      name="email"
      placeholder="juan@correo.com"
      required>
  </div>

  
  <div class="campo">
    <label for="mensaje">Mensaje</label>
    <textarea
      id="mensaje"
      name="mensaje"
      rows="5"
      placeholder="Escribe tu mensaje...">
    </textarea>
  </div>

  
  <button type="submit">Enviar</button>

</form>
1

<form action="/enviar" method="POST"> action define a qué URL se envían los datos al hacer submit. method="POST" envía los datos en el cuerpo de la petición (seguro para contraseñas). Usar GET los pondría en la URL.

1b

novalidate Desactiva la validación nativa del navegador. Los popups de error del navegador son feos y difíciles de personalizar. Con novalidate tomamos el control y aplicamos nuestros propios estilos de validación con CSS y JS.

3

<label for="nombre"> El for debe coincidir con el id del input. Esta asociación hace que al hacer clic en el label, el cursor salte al input correspondiente. También es crítico para lectores de pantalla.

4

type="text" El tipo determina el comportamiento: text = texto libre, email = valida formato email, password = oculta caracteres, number = solo números, tel = abre teclado numérico en móvil.

4b

required Atributo booleano de validación. Marca el campo como obligatorio. El navegador (o JavaScript) impedirá el envío si está vacío. minlength="2" añade validación de longitud mínima.

4c

name="nombre" El identificador que el servidor usará para leer el valor. Es el nombre de la "variable" en el servidor: si usas PHP, accedes con $_POST['nombre']. Sin name, el campo no se envía.

6

<textarea rows="5"> Para texto largo de múltiples líneas. A diferencia de <input>, no es auto-cerrado — el valor del placeholder va como atributo. rows define la altura inicial en número de líneas.

7

<button type="submit"> Envía el formulario. Si no especificas type, el comportamiento por defecto de un botón dentro de un <form> ya es submit. Usa type="button" para botones que no deben enviar el form.

Código CSS — Los Estados Visuales

Estilos del Input y Pseudo-clases

formulario.css 
/* ── CAMPO BASE ──────────────────────────── */
input, textarea, select {
  width: 100%;              /* [A] Ocupa todo el ancho del contenedor */
  padding: 0.75rem 1rem;   /* [B] Espacio interno */
  border: 1.5px solid #334155; /* [C] Borde gris por defecto */
  border-radius: 0.625rem; /* [D] Bordes redondeados */
  background: #0f172a;    /* [E] Fondo oscuro */
  color: #f8fafc;          /* [F] Texto claro */
  font-size: 0.9rem;
  outline: none;           /* [G] Elimina el outline del navegador */
  transition: border-color 0.25s, box-shadow 0.25s; /* [H] */
}

/* ── ESTADO: FOCO (usuario está escribiendo) ── */
input:focus, textarea:focus {
  border-color: #10b981;  /* [I] Borde verde al enfocar */
  box-shadow: 0 0 0 3px rgba(16,185,129,0.15); /* [J] */
}

/* ── ESTADO: INVÁLIDO (valor incorrecto) ────── */
input:invalid:not(:placeholder-shown) {
  border-color: #f87171;  /* [K] Borde rojo si el valor es inválido */
  box-shadow: 0 0 0 3px rgba(248,113,113,0.15);
}

/* ── ESTADO: VÁLIDO (valor correcto) ─────────── */
input:valid:not(:placeholder-shown) {
  border-color: #34d399;  /* [L] Borde verde si el valor es correcto */
}

/* ── PLACEHOLDER ─────────────────────────── */
input::placeholder {
  color: #475569;          /* [M] Texto de ejemplo más tenue */
}

/* ── LABEL ────────────────────────────────── */
label {
  display: block;          /* [N] Ocupa toda la línea */
  font-size: 0.85rem;
  font-weight: 600;
  color: #94a3b8;
  margin-bottom: 0.4rem;  /* [O] Separa el label del input */
}
G

outline: none Elimina el contorno azul/naranja que el navegador agrega al enfocar. Importante: siempre reemplázalo con un estilo propio (border-color + box-shadow) para mantener la accesibilidad visual.

H

transition: border-color 0.25s, box-shadow 0.25s Anima los cambios de estado. Al pasar de default → focus o de válido → inválido, el cambio visual será suave. Transiciones múltiples se separan con coma.

J

box-shadow: 0 0 0 3px rgba(16,185,129,0.15) Crea un "anillo" de enfoque de 3px alrededor del campo. Los 4 valores son: offset-x | offset-y | blur | spread-radius. Con blur=0 y spread=3px, la sombra es un borde sólido exterior.

K

:invalid:not(:placeholder-shown) Pseudo-clase compuesta. :invalid se activa cuando el valor no cumple las validaciones (required, email format, etc.). :not(:placeholder-shown) excluye los campos vacíos, para no mostrar el error antes de que el usuario escriba.

L

:valid:not(:placeholder-shown) Lo opuesto al anterior. Se activa cuando el valor cumple todas las validaciones HTML. El :not(:placeholder-shown) evita mostrar el borde verde en campos vacíos no requeridos.

M

::placeholder Pseudo-elemento para estilizar el texto de ejemplo. El placeholder debe ser significativamente más tenue que el texto real para que el usuario sepa que no ha escrito nada todavía. Dos colons (::) indica pseudo-elemento.

N

display: block en label Por defecto, el label es inline (como texto). Con block, ocupa toda la línea y el input queda debajo. Si no, label e input aparecerían lado a lado en la misma línea.
