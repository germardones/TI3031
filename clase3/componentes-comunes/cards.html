Variante 1 — Feature Card

Tarjeta de Característica

card-feature.html 

<div class="card-feature">

  
  <div class="card-icon-box">⚡</div>

  
  <h3>Rendimiento</h3>
  <p>Optimizado para cargar en menos de 2 segundos.</p>

  
  <a href="#" class="card-link">Saber más →</a>

</div>
card-feature.css 
.card-feature {
  display: flex;                    /* [A] */
  flex-direction: column;          /* [B] */
  gap: 1rem;                       /* [C] */
  padding: 2rem;
  border-radius: 1.25rem;          /* [D] */
  border: 1px solid rgba(255,255,255,0.08); /* [E] */
  transition: all 0.3s ease;
}
.card-feature:hover {
  transform: translateY(-8px);     /* [F] */
  border-color: #10b981;
  box-shadow: 0 20px 40px -15px rgba(16,185,129,0.3); /* [G] */
}

.card-icon-box {
  width: 48px;
  height: 48px;
  border-radius: 0.75rem;          /* [H] */
  background: rgba(16,185,129,0.15); /* [I] */
  display: flex;
  align-items: center;
  justify-content: center;
}
A

display: flex Activa Flexbox en la tarjeta. Convierte sus hijos (ícono, título, párrafo, enlace) en flex-items que se pueden organizar con precisión.

B

flex-direction: column Cambia el eje principal a vertical. Por defecto Flexbox apila los elementos en fila (row). Con column los apilamos hacia abajo, que es lo que queremos en una tarjeta.

C

gap: 1rem Espacio automático entre hijos. En vez de agregar margin-bottom a cada elemento, gap aplica el espacio entre todos los hijos automáticamente, sin afectar el primero ni el último.

D

border-radius: 1.25rem Esquinas redondeadas. Las tarjetas modernas usan radios generosos (12-20px). Transmite suavidad y modernidad. En cambio, border-radius: 0 comunica rigidez y formalidad.

E

border: 1px solid rgba(255,255,255,0.08) Borde sutil. Un borde muy transparente (8% de opacidad) define los límites de la tarjeta sin hacerla pesada visualmente. Es la técnica glassmorphism.

F

transform: translateY(-8px) Efecto "levitar" en hover. Mueve la tarjeta 8px hacia arriba al pasar el cursor, dando feedback visual de interactividad. transform no afecta el layout — los elementos vecinos no se mueven.

G

box-shadow: 0 20px 40px -15px rgba(…,0.3) Sombra de profundidad con color. El cuarto valor negativo (-15px) es el spread negativo: contrae la sombra para que no se extienda hacia los lados, solo hacia abajo. Crea una sombra más realista.

H

border-radius: 0.75rem en la caja del ícono. Esquinas redondeadas pero no circulares. Diferencia entre la caja del ícono (cuadrada-redondeada) y la tarjeta (más redondeada), creando jerarquía visual.

I

background: rgba(16,185,129,0.15) Fondo de color con baja opacidad. El mismo color del acento al 15% de transparencia. Crea una "mancha de color" suave que destaca el ícono sin competir con el texto principal.

Tarjeta de Perfil

<div class="card-profile">

  
  <div class="profile-cover"></div>

  <div class="profile-body">

    
    <div class="profile-avatar">👩‍💻</div>

    <h3>Ana García</h3>

    <div class="profile-stats">

      </div>
    </div>

  </div>

card-profile.css 
.card-profile {
  border-radius: 1.25rem;
  overflow: hidden;              /* [A] */
}

.profile-cover {
  height: 100px;
  background: linear-gradient(135deg, #10b981, #3b82f6); /* [B] */
}

.profile-avatar {
  width: 64px;
  height: 64px;
  border-radius: 50%;           /* [C] */
  border: 3px solid #1e293b;   /* [D] */
  margin-top: -32px;            /* [E] */
}

.profile-stats {
  display: flex;
  gap: 1.5rem;
  border-top: 1px solid rgba(255,255,255,0.06); /* [F] */
  padding-top: 1rem;
}

.stat {
  display: flex;
  flex-direction: column;       /* [G] */
  align-items: center;
}
A

overflow: hidden en la tarjeta. Recorta todo lo que salga de los bordes redondeados. Sin esto, el banner de color y el avatar ignorarían el border-radius de la tarjeta y "saldrían" de las esquinas. Es clave cuando hay hijos que deben respetar el borde.

B

linear-gradient(135deg, #10b981, #3b82f6) Degradado diagonal. El primer valor es el ángulo (135° = diagonal de arriba-izquierda a abajo-derecha). Le da personalidad al banner usando los colores de la marca sin necesitar una imagen.

C

border-radius: 50% Convierte el avatar en círculo perfecto. Al ser 50% del ancho/alto, todos los vértices del cuadrado quedan dentro del círculo. Sólo funciona perfectamente en elementos cuadrados (width = height).

D

border: 3px solid #1e293b en el avatar. Separación visual entre avatar y banner. El borde del mismo color que el fondo de la tarjeta crea un efecto de "espacio" alrededor del avatar, separándolo del cover. Se llama ring o outline trick.

E

margin-top: -32px El truco del avatar superpuesto. El avatar mide 64px de alto. Con margin-top: -32px (la mitad) sube hacia arriba y se superpone al banner. Los márgenes negativos sacan al elemento de su posición natural, solapándose con el elemento anterior.

F

border-top en .profile-stats. Separador sutil entre el contenido y las estadísticas. En vez de usar un <hr>, se aplica un borde superior directamente al contenedor de stats.

G

flex-direction: column en .stat. Apila el número encima de la etiqueta. Cada stat tiene el número (grande) arriba y la etiqueta (pequeña) abajo. Flex column + align-items: center lo centra todo automáticamente.

Variante 3 — Pricing Card

Tarjeta de Precio

card-pricing.html 
<div class="card-pricing featured">

  
  <span class="pricing-badge">Popular</span>

  
  <p class="plan-name">Pro</p>
  <div class="price">$19<span>/mes</span></div>
  <p class="price-desc">Todo lo que necesitas</p>

  
  <ul>
    <li>5 proyectos activos</li>
    <li class="disabled">Análisis avanzado</li>
  </ul>

  
  <button class="btn-plan primary">Empezar ahora</button>

</div>
card-pricing.css 
.card-pricing {
  display: flex;
  flex-direction: column;
  position: relative;           /* [A] */
  overflow: hidden;
  border-radius: 1.25rem;
  padding: 2rem;
}

.card-pricing.featured {
  border-color: #10b981;
  box-shadow: 0 0 0 1px #10b981, /* [B] */
              0 20px 40px -15px rgba(16,185,129,0.3);
}

.pricing-badge {
  position: absolute;           /* [C] */
  top: 1rem;
  right: 1rem;
}

.price { font-size: 2.75rem; }   /* [D] */
.price span { font-size: 1rem; } /* [E] */

.card-pricing ul {
  flex-grow: 1;                 /* [F] */
  list-style: none;
}

.card-pricing ul li {
  display: flex;
  align-items: center;
  gap: 0.5rem;
}
.card-pricing ul li::before {
  content: "✓";                /* [G] */
  color: #10b981;
}
.card-pricing ul li.disabled::before {
  content: "×";                /* [H] */
  color: #475569;
}
A

position: relative en la tarjeta. Crea un contexto de posicionamiento. El badge ("Popular") necesita position: absolute para ubicarse en la esquina. Para que funcione, el padre debe tener position: relative, de lo contrario el badge se posicionaría respecto al viewport completo.

B

box-shadow: 0 0 0 1px #10b981, 0 20px… Doble sombra con coma. Se pueden apilar múltiples sombras separadas por coma. La primera (0 0 0 1px) es un borde simulado con sombra (sin desplazamiento, solo spread). La segunda es la sombra de profundidad.

C

position: absolute; top: 1rem; right: 1rem en el badge. Sale del flujo normal del documento y se posiciona en coordenadas relativas al padre. No empuja a ningún otro elemento — flota encima del contenido.

D / E

Tamaños del precio. Jerarquía tipográfica: el número ($19) es grande (2.75rem ≈ 44px) para ser el foco visual. La unidad (/mes) es pequeña (1rem) para no competir. Este contraste guía el ojo hacia la información más importante.

F

flex-grow: 1 en la lista de features. Llena el espacio disponible. En un grid de tarjetas de precio, cada tarjeta puede tener distinto número de items. flex-grow: 1 hace que la lista ocupe todo el espacio libre, empujando el botón siempre al fondo — así todos los botones quedan alineados.

G

li::before { content: "✓" } Pseudo-elemento que inyecta contenido antes de cada ítem. No necesitas escribir el ✓ en el HTML. CSS lo agrega automáticamente a todos los <li>, lo que hace el código más limpio y fácil de cambiar.

H

li.disabled::before { content: "×" } Override del pseudo-elemento para items desactivados. Al agregar la clase .disabled al <li>, se sobreescribe el ::before por defecto. El mismo mecanismo, diferente resultado — elegancia CSS.

¿Cuándo usar cada variante?

🃏 Feature Card — Para mostrar servicios, características de producto o beneficios en una sección "¿Por qué elegirnos?".

👤 Profile Card — Para equipos, autores de blog, testimonios o directorios de personas.

💰 Pricing Card — Para planes de suscripción, comparativas de productos o niveles de servicio.

Las tres comparten la misma base: display:flex + flex-direction:column + border-radius. La diferencia está en qué hijos contienen y cómo se organizan.
