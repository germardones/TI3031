Closures: Memoria Privada de Funciones

Un closure ocurre cuando una función interna captura variables del scope léxico externo. Esas variables permanecen vivas mientras el closure exista.

Contador con estado privado

function createCounter() {
  let count = 0; // Variable privada
  return {
    increment: () => ++count,
    decrement: () => --count,
    value: () => count
  };
}

const c = createCounter();
c.increment(); // 1
c.increment(); // 2
console.log(c.value()); // 2
// count no es accesible desde fuera
Función Fábrica (Factory Function)

Crea múltiples instancias con estado independiente. Cada llamada genera un nuevo closure separado.

function makeMultiplier(factor) {
  return (n) => n * factor;
}
const doble = makeMultiplier(2);
const triple = makeMultiplier(3);
doble(5); // 10
triple(5); // 15
Closures en el mundo real
  • • React useState: el setter recuerda a qué componente pertenece
  • • Middleware Express: captura configuración de la app
  • • Event listeners: acceden a variables del scope donde se crearon
  • • Módulos JS: variables privadas del módulo

Funciones Puras e Impuras

Una función pura es predecible y no modifica el mundo exterior. Son la base de la programación funcional y facilitan enormemente el testing.

Característica✅ Función Pura⚠️ Función Impura
Mismo inputSiempre mismo outputPuede variar (depende de estado externo)
Efectos secundariosNingunoModifica variables externas, BD, DOM, etc.
TestingTrivial — solo verificar retornoNecesita mocks, spies, setup/teardown
DepuraciónMuy fácilCompleja — depende del contexto
Cuándo usarTransformar datos, cálculos, filtrosInteracción con APIs, DOM, DB, logs
✅ Pura
const calcIVA = (precio, tasa) =>
  precio * (1 + tasa);

calcIVA(100, 0.19); // Siempre 119
⚠️ Impura
let tasa = 0.19;
const calcIVA = (precio) =>
  precio * (1 + tasa); // Depende de tasa externa

tasa = 0.21; // Ahora calcIVA(100) = 121

Composición y Currying

Currying

Transforma una función de N argumentos en N funciones de 1 argumento. Permite aplicación parcial — pre-configurar funciones con algunos parámetros.

const sumar = (a) => (b) => a + b;

const sumar10 = sumar(10);
sumar10(5); // 15
sumar10(20); // 30

// Útil para crear variantes de una función base
const addTax = (rate) => (price) => price * (1 + rate);
const addVAT = addTax(0.19);

Composición (pipe)

Encadena funciones donde el resultado de una es el input de la siguiente. Construye transformaciones complejas desde piezas simples y reutilizables.

const pipe = (...fns) => (x) =>
  fns.reduce((v, f) => f(v), x);

const trim = (s) => s.trim();
const lower = (s) => s.toLowerCase();
const slug = (s) => s.replace(/ /g, "-");

const toSlug = pipe(trim, lower, slug);
toSlug(" Hola Mundo "); // "hola-mundo"
Arrow functions y el contexto this

Las arrow functions no tienen su propio this. Capturan el this del scope donde fueron definidas (léxico). Las funciones tradicionales crean su propio this según cómo se llaman.

const obj = {
  nombre: "App",
  traditional: function() { return this.nombre; }, // "App"
  arrow: () => this.nombre // undefined — this = window/undefined
};
