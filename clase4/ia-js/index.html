Prompt Engineering

Prompt engineering es la diferencia entre una IA que da respuestas genéricas y una que actúa como un especialista. Un prompt bien construido tiene 4 elementos.

❌ Prompt Débil
"Haz una función de login"

Sin contexto. La IA adivina el lenguaje, el framework, la lógica y el formato. El resultado será genérico y probablemente inútil en tu proyecto real.

✅ Prompt Fuerte
"Eres un dev senior de Node.js.
Crea una función loginUser(email, password)
que use bcrypt para comparar contraseñas
y JWT para generar tokens.
Maneja errores con try/catch.
Retorna { token, user } o lanza Error.
Sin dependencias extra."
🎯
1. Contexto

Quién eres tú o qué rol debe adoptar la IA. "Eres un senior dev de React con 10 años de experiencia."

📋
2. Tarea

Qué necesitas exactamente. Verbos claros: "Crea", "Refactoriza", "Explica", "Encuentra bugs en".

⚙️
3. Restricciones

Límites y condiciones. "Sin librerías externas", "compatible con ES5", "menos de 20 líneas".

📄
4. Formato

Cómo quieres la respuesta. "En una tabla", "con comentarios JSDoc", "primero el código, luego la explicación".

Chain of Thought

Añade "Piensa paso a paso antes de responder." La IA razona el problema antes de dar código, reduciendo errores lógicos.

Role Prompting

"Actúa como un revisor de código de Google." Ajusta el nivel de detalle, las mejores prácticas y el vocabulario técnico.

Few-Shot

"Dado este input → output de ejemplo, haz lo mismo con este nuevo input." Los ejemplos guían el formato y la lógica esperada.

Limitaciones Reales de la IA

La IA es una herramienta poderosa, pero no infalible. Conocer sus límites es lo que distingue a un profesional de alguien que solo copia y pega.

⚠️ Alucinaciones

La IA puede inventar funciones que no existen, documentación falsa, o afirmar con certeza cosas incorrectas. Siempre verifica en la documentación oficial.

📅 Conocimiento con fecha de corte

Los modelos tienen un corte de entrenamiento. No conocen frameworks muy nuevos ni cambios recientes de APIs. Para tecnología reciente, consulta siempre la documentación oficial.

🔒 Vulnerabilidades de Seguridad

Código generado por IA puede tener XSS, SQL Injection u otras vulnerabilidades si no especificas requisitos de seguridad. Pide explícitamente validación de inputs y manejo seguro de datos.

✅ Para qué sí es excelente
  • • Boilerplate y scaffolding rápido
  • • Generar unit tests y casos de prueba
  • • Explicar código heredado complejo
  • • Sugerir refactorizaciones y mejoras
  • • Documentar funciones y APIs

Flujo de Trabajo Profesional

El desarrollador profesional usa la IA como un pair programmer — no como un reemplazo. El criterio y la responsabilidad siempre son tuyos.

1
Definir el problema claramente

Antes de preguntar a la IA, escribe en palabras propias qué necesitas resolver. Si no puedes explicarlo, no estás listo para pedir ayuda.

2
Construir un prompt con contexto

Incluye: tecnologías, restricciones, ejemplos de lo que esperas. Más contexto = mejor respuesta.

3
Leer y entender el código generado

Nunca copies código sin entenderlo. Pregunta a la IA que te explique cada parte que no entiendas. Eres responsable de lo que sube a producción.

4
Verificar y testear

Prueba con edge cases. Comprueba contra la documentación oficial. El código de la IA puede funcionar en el caso feliz y fallar en los bordes.

5
Iterar y refinar

La primera respuesta raramente es la final. Pide mejoras específicas: "Hazlo más eficiente", "Añade manejo de errores", "Simplifica la lógica del if".
