Haz clic en una línea

El DOM no es estático. Podemos crear, modificar y eliminar elementos en respuesta a datos o acciones del usuario — esto es lo que permite apps como Gmail funcionar sin recargar.

Ciclo de Vida de un Elemento

1. Creación (en memoria)

createElement crea el nodo en la memoria de JS. Aún no es visible — no existe en el árbol del DOM del documento.

const card = document.createElement("div");
card.className = "card";
card.dataset.id = "42";

2. Inyección (visible)

Al usar appendChild o prepend, el navegador detecta el cambio y produce un reflow y un repaint para mostrar el elemento.

container.appendChild(card);
// También: prepend, before, after
// insertAdjacentElement

3. Destrucción

element.remove() extrae el nodo del DOM. Importante: si tiene event listeners activos, pueden causar memory leaks si no se limpian.

card.remove();
// Limpiar listeners antes:
controller.abort();

Selección de Elementos

querySelector y querySelectorAll aceptan cualquier selector CSS. Son los métodos modernos; los métodos como getElementById quedan para casos donde el rendimiento es crítico.

MétodoRetornaCuándo usar

querySelectorAll(css)NodeList (array-like)Múltiples elementos — iterar con forEach

// Selección básica
const btn = document.querySelector("#submit");
const inputs = document.querySelectorAll(".form-field");
inputs.forEach(input => input.setAttribute("required", ""));

// Subir en el árbol desde un hijo
item.addEventListener("click", e => {
  const card = e.target.closest(".card");
  card?.remove();
});

Eventos y Delegación de Eventos

El objeto Event

Cada listener recibe un objeto Event con información del suceso. Propiedades clave para manejar correctamente cada interacción.

btn.addEventListener("click", (e) => {
  e.preventDefault(); // No recargar/navegar
  e.stopPropagation(); // No burbujear
  console.log(e.target); // Elemento clickeado
  console.log(e.currentTarget); // El que escucha
});

Delegación de Eventos

Un listener en el padre cubre todos los hijos, incluso los que se crean dinámicamente. Más eficiente — un solo listener en vez de cien.

lista.addEventListener("click", (e) => {
  if (e.target.matches(".delete-btn")) {
    e.target.closest("li")?.remove();
  }
  if (e.target.matches(".edit-btn")) {
    // Manejo de edición
  }
});
AbortController: limpiar listeners

Permite cancelar múltiples event listeners a la vez pasando la misma señal. Evita memory leaks al desmontar componentes.

const controller = new AbortController();
btn.addEventListener("click", handler, { signal: controller.signal });
input.addEventListener("keydown", handler2, { signal: controller.signal });
// Cancela TODOS los listeners registrados con esta señal:
controller.abort();

localStorage: Persistencia en el Cliente

localStorage almacena datos como texto en el dispositivo del usuario. Persiste entre sesiones (a diferencia de sessionStorage que se borra al cerrar la pestaña).

// Guardar (solo strings)
localStorage.setItem("theme", "dark");

// Guardar objetos — serializar a JSON
localStorage.setItem("user", JSON.stringify(userObj));

// Leer
const theme = localStorage.getItem("theme");
const user = JSON.parse(localStorage.getItem("user"));

// Eliminar
localStorage.removeItem("theme");
localStorage.clear(); // Todo

⚠️ Advertencia de Seguridad

Nunca guardes tokens de sesión, contraseñas ni datos sensibles en localStorage. Es accesible desde cualquier script de la página — un ataque XSS puede robarlos fácilmente.
