El DOM no es estático. Podemos crear, modificar y eliminar elementos basándonos en datos. Esta capacidad es la que permite que aplicaciones como Gmail funcionen sin recargar la página.

Ciclo de Vida de un Elemento

1. Creación

El elemento nace en la memoria de JS mediante createElement, pero aún no es visible en el navegador.

2. Inyección

Al usar appendChild o prepend, el navegador detecta el cambio en el árbol del DOM y lo muestra visualmente.

3. Destrucción

Mediante remove(), eliminamos el nodo. Es vital para limpiar la memoria en apps grandes.
