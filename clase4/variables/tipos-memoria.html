Gestión de Memoria y Tipado

Profundización técnica: de la asignación escalar a la recolección de basura generacional.

1. Catálogo de Tipos en JavaScript

JavaScript es un lenguaje de tipado dinámico, donde las variables no tienen un tipo fijo, pero los valores sí.

Primitivos (By Value)

let num = 42; // Number
let str = "Pro"; // String
let bool = true; // Boolean
let empty = null; // Objeto nulo intencional
let undef; // Undefined (no asignado)
let big = 9007n; // BigInt
let sym = Symbol(); // Identificador único

Estructuras (By Reference)

// Objetos (Colecciones de datos)
const user = { id: 1, name: "Ana" };

// Arreglos (Listas ordenadas)
const list = [10, 20, 30];

// Funciones (Bloques ejecutables)
const run = () => true;

2. Estructuras Dinámicas: LIFO vs FIFO

La memoria en JS no solo se asigna, sino que se organiza bajo reglas lógicas de acceso.

LIFO (The Stack)

"Last In, First Out": El último en entrar es el primero en salir. Como una pila de platos.

Uso: Gestión de llamadas a funciones.

FIFO (The Queue)

"First In, First Out": El primero en entrar es el primero en salir. Como una fila en el banco.

Uso: Event Loop y gestión de tareas asíncronas.

3. Punteros y Referencias

¿Qué es un Puntero?

Un puntero es una variable cuyo único propósito es almacenar la dirección de memoria de otra ubicación donde se encuentran los datos reales.

En lenguajes como C++, los punteros son explícitos. En JavaScript, son implícitos y los llamamos Referencias. Cuando asignas un objeto a una variable, no estás guardando el objeto, sino un puntero que le dice a JavaScript dónde buscarlo en el Heap.

Variable (Stack) #0x0A01

"He guardado la dirección #0x0A01. Si el motor me pide el valor, iré al Heap a buscarlo."

Data (Heap) #0x0A01

{ status: "OK", value: 512 }

4. El Motor de Recolección (GC)

JavaScript utiliza una recolección de basura de tipo **Generacional**, dividiendo el Heap en dos áreas principales para optimizar el rendimiento.

Young Generation

Es un espacio pequeño donde se alojan objetos nuevos. El recolector pasa muy a menudo aquí (Minor GC) porque la mayoría de los objetos mueren jóvenes (objetos temporales en funciones).

Old Generation

Si un objeto sobrevive varios ciclos de recolección en el Young Space, se promueve al Old Space. Aquí la recolección es menos frecuente (Major GC) pero mucho más pesada.

Fases del Algoritmo Mark-and-Sweep
1

Root Scanning: El GC identifica todos los objetos "raíz" (global object like window, variables actuales en el stack).

2

Marking: Recorre recursivamente todas las referencias. Cada objeto alcanzado es marcado con un "bit" de vida.

3

Sweeping: Se recorre todo el Heap. Cualquier objeto sin la marca de vida es eliminado y su espacio de memoria se reclama como libre.
