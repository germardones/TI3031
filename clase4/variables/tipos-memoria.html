¿Qué es una Variable?

La variable es el concepto más fundamental de la programación. Todo lo demás se construye sobre esto.

1. La Caja Etiquetada

Imagina que tu programa es un cuarto lleno de cajas etiquetadas. Cada caja tiene un nombre (la etiqueta) y guarda algo dentro (el valor).

Una variable es exactamente eso: un espacio con nombre que guarda un dato.

Así se ven las variables en memoria (de forma simplificada)

nombre
"Ana"
texto (string)
edad
22
número (number)
estaActivo
true
booleano (boolean)
sinUsar
undefined
sin valor asignado
// Declarar = crear la caja y ponerle etiqueta
// Asignar = meter algo dentro de la caja

let nombre = "Ana"; // caja "nombre" → guarda "Ana"
let edad = 22; // caja "edad" → guarda 22
let estaActivo = true; // caja "estaActivo" → guarda true
let sinUsar; // caja creada, pero vacía → undefined

2. Declarar vs. Asignar

Son dos pasos distintos. Puedes hacerlos juntos o por separado.

Declarar

Le dices al programa "existe una caja con este nombre". Usas una palabra reservada: let, const, o var.

let puntuacion; // caja creada, vacía
console.log(puntuacion); // undefined
Asignar

Le metes un valor a la caja usando el operador =. Puedes hacerlo al momento de declarar o después.

let puntuacion; // declarar
puntuacion = 100; // asignar

// O todo junto:
let vidas = 3;

El = no significa "igual", significa "guardar"

En matemáticas x = 5 dice que x es 5. En JavaScript dice "mete el valor 5 dentro de la caja x". Por eso x = x + 1 tiene sentido: toma el valor actual, súmale 1, y guarda el resultado de nuevo.

3. let, const y var

Hay tres formas de declarar una variable. Cada una tiene un comportamiento diferente. Por ahora, la regla práctica:

Regla práctica

Usa const por defecto. Cambia a let si necesitas reasignar. Evita var.

🔒

const

La caja queda sellada: no puedes cambiar qué guarda. Ideal para valores que no cambian.

const PI = 3.14159;
const sitio = "INACAP";

// Esto lanza error:
PI = 3; // ❌ TypeError
🔄

let

La caja se puede vaciar y rellenar. Úsalo cuando el valor va a cambiar.

let puntos = 0;
puntos = 10; // ok
puntos = puntos + 5; // ok

// puntos ahora vale 15
⚠️

var

La forma antigua. Tiene comportamientos raros (hoisting) que generan bugs. No la uses en código nuevo.

// ⚠ Evitar en código moderno
var viejo = "anticuado";

// Usa let o const en su lugar

4. Cómo Nombrar Variables

JavaScript tiene reglas para los nombres. Además, hay convenciones que hacen el código más legible.

Reglas obligatorias

✓

Puede empezar con letra, _ o $

✓

Puede tener números (pero no al inicio)

✗

No puede empezar con número: 1nombre

✗

No puede tener espacios: mi nombre

✗

No puede ser una palabra reservada: let, if, for...

Convenciones de estilo

JavaScript usa camelCase para variables y funciones.

// Bien — camelCase
let nombreUsuario = "Ana";
let precioTotal = 1500;

// Evitar — confuso
let NombreUsuario = "Ana"; // PascalCase (para clases)
let nombre_usuario = "Ana"; // snake_case (Python, no JS)

Nombres descriptivos hacen código legible

// Mal: nadie sabe qué significa esto
let x = 86400;

// Bien: se entiende sin comentario
const SEGUNDOS_POR_DIA = 86400;

5. Leer el Tipo: typeof

JavaScript es de tipado dinámico: no debes declarar el tipo, JavaScript lo detecta solo. Puedes consultarlo con typeof.

let nombre = "Carlos";
let edad = 25;
let activo = true;
let nada;

console.log(typeof nombre); // "string"
console.log(typeof edad); // "number"
console.log(typeof activo); // "boolean"
console.log(typeof nada); // "undefined"

El tipo de una variable puede cambiar porque en JavaScript la variable no tiene tipo fijo, el valor sí lo tiene. Esto se llama tipado dinámico.

let dato = 42;
console.log(typeof dato); // "number"

dato = "ahora soy texto";
console.log(typeof dato); // "string" — cambió el tipo
