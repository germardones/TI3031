Gestión de Memoria y Tipado

Del valor primitivo al motor V8: cómo JavaScript gestiona cada byte de tu programa.

1. Catálogo de Tipos en JavaScript

JavaScript es un lenguaje de tipado dinámico. Las variables no tienen tipo fijo — los valores sí.

Primitivos (By Value)

Se copian por valor: cada variable tiene su propia copia independiente en el Stack.

let num = 42; // Number
let str = "Pro"; // String
let bool = true; // Boolean
let empty = null; // Nulo intencional
let undef; // Undefined
let big = 9007n; // BigInt
let sym = Symbol(); // ID único
Estructuras (By Reference)

Se pasan por referencia: la variable guarda una referencia (puntero) al objeto en el Heap.

// Objetos
const user = { id: 1, name: "Ana" };

// Arreglos
const list = [10, 20, 30];

// Funciones
const run = () => true;

2. Estructuras Dinámicas: LIFO y FIFO

JavaScript no guarda datos aleatoriamente. Usa dos reglas de acceso fundamentales según el contexto de ejecución.

LIFO — El Call Stack

Cada vez que llamas a una función, JS apila un nuevo execution context encima. Cuando la función termina, se extrae de la cima.

// A llama B, B llama C
// Stack: [Global → A → B → C]
// C termina: [Global → A → B]
// B termina: [Global → A]

⚠ Stack Overflow

Si una función se llama a sí misma sin condición de parada (recursión infinita), la pila supera su límite y el motor lanza RangeError: Maximum call stack size exceeded.

FIFO — El Event Loop

Las operaciones asíncronas (setTimeout, fetch) se encolan en una cola FIFO. El Event Loop las procesa en orden cuando el Call Stack está vacío.

// Macrotask Queue (FIFO)
setTimeout(() => console.log("A"), 0);
setTimeout(() => console.log("B"), 0);
// Siempre: A luego B

// Microtask Queue (mayor prioridad)
Promise.resolve().then(() => console.log("Micro"));

3. El Heap: Memoria Dinámica

El Heap es la memoria donde viven todos los objetos de tu programa. A diferencia del Stack, no tiene un tamaño fijo ni un orden de acceso rígido.

Característica Stack (Pila) Heap (Montón)
¿Qué guarda? Primitivos, referencias, marcos de función Objetos, arreglos, funciones
Tamaño Fijo y pequeño (~1-8 MB según entorno) Dinámico y grande (límite del SO)
Velocidad Muy rápido (acceso directo) Más lento (necesita seguir puntero)
Liberación Automática al salir de la función El Garbage Collector decide cuándo

4. Punteros y Referencias

El problema de las referencias compartidas

Cuando asignas un objeto a otra variable, ambas apuntan al mismo lugar en el Heap. Modificar uno cambia el otro — un error común para quien viene de trabajar con primitivos.

const a = { valor: 1 };
const b = a; // b apunta al MISMO objeto
b.valor = 99;
console.log(a.valor); // 99 — a también cambió!

// Solución: clonar para romper la referencia
const c = { ...a }; // copia superficial
const d = structuredClone(a); // copia profunda

5. El Motor de Recolección (GC)

JavaScript gestiona la memoria automáticamente. El Garbage Collector usa un algoritmo llamado Mark-and-Sweep para identificar y liberar objetos sin referencias activas.

Young Generation

Objetos nuevos nacen aquí. El GC visita con alta frecuencia porque la mayoría de objetos temporales (variables de funciones) mueren en su primera o segunda iteración.

Old Generation

Si un objeto sobrevive varios ciclos, se "promueve" al Old Space. Aquí el GC es menos frecuente pero más costoso. Un Major GC puede causar micro-pausas perceptibles.

Algoritmo Mark-and-Sweep
1

Root Scanning: El GC identifica todos los objetos raíz: variables globales, variables activas en el Stack.

2

Marking: Recorre todas las referencias recursivamente. Cada objeto alcanzado recibe una marca de "vivo".

3

Sweeping: Todo objeto sin marca = sin referencias = memoria libre. Se recicla el espacio.

6. El Motor V8 de JavaScript

V8 es el motor que ejecuta tu código. No interpreta JavaScript línea por línea — lo compila a código máquina de forma inteligente usando una técnica llamada JIT (Just-In-Time).

El código fuente se transforma en un árbol de sintaxis abstracta. Aquí se detectan errores de sintaxis antes de ejecutar.

Ignition

Intérprete que convierte el AST a bytecode. Rápido de generar. También recoge datos sobre qué funciones se llaman más.

TurboFan

Para funciones "calientes" (frecuentes), TurboFan genera código máquina nativo. Usa Hidden Classes e Inline Caching para optimizar al máximo.
