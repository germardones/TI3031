Protección de Estructuras

Estrategias avanzadas para la preservación de la integridad del estado y el control de descriptores.

1. Reasignación vs Mutabilidad

Es fundamental distinguir entre la protección del identificador en el Stack y la protección del contenido en el Heap.

Pila (Stack)

Impide que la variable apunte a una dirección de memoria diferente. Esta es la función principal de const.

const x = [1];
x = [2]; // Error: Assignment to const

Montón (Heap)

Los valores internos del objeto siguen siendo modificables. const NO protege las propiedades internas.

const x = [1];
x.push(2); // Operación permitida

2. Property Descriptors

Para un control granular del motor V8, podemos definir metadatos sobre cada propiedad mediante descriptores.

  • • Writable: ¿El valor puede ser alterado?
  • • Enumerable: ¿Aparece en bucles como for...in?
  • • Configurable: ¿Se puede borrar o redefinir el descriptor?
Object.defineProperty(obj, 'secret', {
  value: 999,
  writable: false,
  configurable: false
});

3. Estrategias de Congelación

Object.seal()

"Sella" el objeto: no se pueden añadir ni borrar propiedades, pero las existentes siguen siendo editables si eran mutables.

// Estructura fija, valores editables.

Object.freeze()

Inmutabilidad total (superficial). El objeto se convierte en solo lectura. Ideal para configuraciones de sistema invariables.

// Estructura fija, valores bloqueados.
