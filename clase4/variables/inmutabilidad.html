Scope y Contexto

¿Desde dónde puedes ver una variable? Eso es el scope. Es la pregunta más importante al trabajar con variables.

1. ¿Qué es el Scope?

Scope es el alcance de una variable: desde qué partes del código puedes leerla o modificarla.

Imagina el scope como habitaciones dentro de una casa. Lo que está en el pasillo (global) lo ven todos. Lo que está dentro de una habitación (local) solo lo ve quien está en esa habitación.

Scope Global — visible desde cualquier parte
const app = "MiApp";
Scope de Función — solo visible dentro de esta función
let usuario = "Ana"; // solo existe aquí
Scope de Bloque — solo dentro de este if/for
const temporal = true; // solo existe aquí
const app = "MiApp"; // global: todos lo ven

function saludar() {
  let usuario = "Ana"; // local: solo existe dentro de saludar()
  console.log(app); // ok — puede ver el global
  console.log(usuario); // ok — está en su propio scope
}

saludar();
console.log(app); // ok
console.log(usuario); // ❌ ReferenceError: usuario no existe aquí

2. var, let y const: sus diferencias reales

La diferencia clave entre var, let y const está en su scope y si permiten reasignación.

let — scope de bloque

let

Existe solo dentro del bloque {} donde fue declarada. Puede ser reasignada.

let contador = 0;
contador = 1; // ok — reasignación permitida

if (true) {
  let dentro = "solo aquí";
  console.log(dentro); // ok
}
console.log(dentro); // ❌ ReferenceError — fuera del bloque
const — scope de bloque + no reasignable

const

Igual que let, pero la referencia no puede cambiar. Para objetos y arreglos, su contenido sí puede modificarse.

const nombre = "Ana";
nombre = "Luis"; // ❌ TypeError — no se puede reasignar

// Con objetos: la variable no cambia, el contenido sí
const user = { nombre: "Ana" };
user.nombre = "Luis"; // ok — modificar propiedad
user = {}; // ❌ TypeError — no puedes reemplazar el objeto

const no congela el objeto, solo evita que la variable apunte a otro lugar. El contenido del objeto puede cambiar.

var — scope de función (evitar)

var (el problemático)

var ignora los bloques {} — su scope es la función completa, no el bloque. Esto causa bugs difíciles de encontrar.

if (true) {
  var x = "escapé del bloque";
}
console.log(x); // "escapé del bloque" ← var se sale del if

if (true) {
  let y = "me quedo aquí";
}
console.log(y); // ❌ ReferenceError ← let se queda en el bloque

Resumen comparativo

Scope Reasignable Usar en código moderno
const Bloque {} No Sí — por defecto
let Bloque {} Sí Sí — cuando necesites cambiar el valor
var Función Sí No — comportamiento impredecible

3. Hoisting: el comportamiento raro de var

Hoisting es por qué var puede causar bugs. JavaScript procesa las declaraciones antes de ejecutar el código.

var — hoisting peligroso

console.log(x); // undefined — no da error!
var x = 5;
console.log(x); // 5

// JavaScript lo ve así internamente:
var x; // declaración sube
console.log(x); // undefined
x = 5; // asignación se queda

let/const — error claro

console.log(y); // ❌ ReferenceError
// let/const están en la
// "zona muerta temporal"
// hasta esta línea
let y = 5;
console.log(y); // 5

El error es mejor que el silencio. Con let/const sabes exactamente dónde está el problema.

4. Guía Práctica: ¿Cuál uso?

Con estos tres criterios puedes tomar la decisión correcta siempre:

1

¿El valor va a cambiar? No → usa const

Si declaras algo y no necesitas cambiarlo, usa const. Hace el código más predecible y comunica intención.

const URL_API = "https://api.ejemplo.com";
const MAX_INTENTOS = 3;
const usuario = { nombre: "Ana" }; // const aunque sea objeto
2

¿El valor va a cambiar? Sí → usa let

Contadores, acumuladores, estados que cambian con el tiempo.

let puntos = 0;
let intentos = 3;
let estaActivo = true;

puntos += 10; // suma y reasigna
3

¿Código legado con var? → migra a let/const

Si ves var en código que estás editando, cámbialo. El comportamiento de let/const es más claro y predecible.

Ejercicio mental: ¿qué imprime este código?

const mensaje = "global";

function mostrar() {
  let mensaje = "local"; // variable distinta, mismo nombre
  console.log(mensaje);
}

mostrar();
console.log(mensaje);
Ver respuesta

Imprime "local" y luego "global".
Dentro de mostrar(), el let mensaje local sombrea al global. Fuera de la función, solo existe el const mensaje = "global".
