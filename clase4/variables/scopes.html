Tipos de Datos

Los datos tienen diferentes formas: números, textos, verdadero/falso y más. Conocer cada uno es esencial.

1. Panorama General

JavaScript maneja tipos primitivos y los objetos. Los primitivos son los bloques básicos; los objetos permiten agrupar datos.

Tipo Ejemplo typeof devuelve Para qué sirve
number 42, 3.14, -7 "number" Cantidades, precios, edades, cálculos
string "Hola", 'Ana' "string" Textos, nombres, mensajes
boolean true, false "boolean" Condiciones, estados (activo/inactivo)
undefined undefined "undefined" Variable declarada sin valor
null null "object" ⚠ Ausencia intencional de valor
object { nombre: "Ana" } "object" Agrupar datos relacionados
array [1, 2, 3] "object" Listas ordenadas de elementos

2. Los Tipos Primitivos

number

Números

JavaScript usa un solo tipo para todos los números: enteros y decimales. Incluye valores especiales como Infinity y NaN.

const entero = 42;
const decimal = 3.14;
const negativo = -100;

// Operaciones básicas
const suma = 10 + 5; // 15
const resto = 10 % 3; // 1 (módulo: lo que sobra al dividir)
const potencia = 2 ** 8; // 256

// Valores especiales
console.log(1 / 0); // Infinity
console.log("abc" * 2); // NaN (Not a Number)

NaN es un número que dice "esto no es un número válido". Aparece cuando intentas hacer una operación matemática con texto.

string

Texto (String)

Cualquier texto va entre comillas: simples '...', dobles "...", o backticks ` `. Los backticks permiten insertar variables directamente con ${variable}.

const nombre = "Carlos";
const apellido = 'Soto';

// Concatenar (unir textos con +)
const completo = nombre + " " + apellido; // "Carlos Soto"

// Template literal (más cómodo)
const saludo = `Hola, ${nombre}!`; // "Hola, Carlos!"

// Propiedades útiles
console.log(nombre.length); // 6 (cantidad de letras)
console.log(nombre.toUpperCase()); // "CARLOS"
console.log(nombre.includes("arl")); // true
boolean

Booleano

Solo dos valores posibles: true o false. Es el resultado de cualquier comparación y controla el flujo del programa.

const mayorDeEdad = true;
const estaLogueado = false;

// Las comparaciones devuelven boolean
console.log(10 > 5); // true
console.log(10 === 10); // true (igual en valor y tipo)
console.log(10 === "10"); // false (diferente tipo)
console.log(10 == "10"); // true (== convierte tipos, evitar)
undefined
null

La diferencia entre null y undefined

Ambos representan "ausencia de valor", pero con intenciones distintas.

undefined — JavaScript no asignó nada

let usuario;
console.log(usuario); // undefined

// JavaScript lo pone solo
// cuando no asignas nada

null — tú decidiste que está vacío

let sesion = null;
// Sesión cerrada intencionalmente

sesion = { usuario: "Ana" };
// Ahora hay sesión activa

Regla simple: undefined lo pone JavaScript automáticamente. null lo pones tú cuando quieres decir "esto no tiene valor ahora".

3. Objetos y Arreglos

Cuando necesitas guardar varios datos juntos, usas objetos o arreglos.

object

Objeto — datos con nombre (clave: valor)

Un objeto agrupa datos relacionados. Cada dato tiene un nombre (propiedad) y un valor.

const estudiante = {
  nombre: "María",
  edad: 20,
  activa: true
};

// Acceder a propiedades
console.log(estudiante.nombre); // "María"
console.log(estudiante["edad"]); // 20

// Modificar una propiedad
estudiante.edad = 21; // ok aunque sea const
array

Arreglo — lista ordenada de elementos

Un arreglo guarda varios valores en orden. Cada elemento tiene una posición (índice) que empieza en 0.

const notas = [6.5, 5.8, 7.0, 6.2];

// Acceder por posición (empieza en 0)
console.log(notas[0]); // 6.5 (primera)
console.log(notas[2]); // 7.0 (tercera)

// Propiedades y métodos útiles
console.log(notas.length); // 4 (cantidad de elementos)
notas.push(6.8); // agrega al final
notas.pop(); // elimina el último

4. Comparar: == vs ===

JavaScript tiene dos formas de comparar. La diferencia puede causar bugs silenciosos.

== Igualdad débil

Compara el valor, pero convierte el tipo si son distintos. Produce resultados inesperados.

console.log(5 == "5"); // true ← peligroso
console.log(0 == false); // true ← confuso
console.log("" == false); // true ← muy confuso

=== Igualdad estricta

Compara valor y tipo. Si no son del mismo tipo, devuelve false. Siempre usa esta.

console.log(5 === "5"); // false ← correcto
console.log(0 === false); // false ← correcto
console.log(5 === 5); // true ← como esperamos

Regla: Siempre usa === para comparar. Nunca ==.
