Operadores de Decisión en Profundidad

Operador OR ||

Retorna el primer valor truthy. Úsalo para valores por defecto, pero cuidado con 0 o cadenas vacías válidas — los trata como falsos.

const n = config.port || 3000;
// Si config.port = 0, usa 3000 (bug!)

Nullish ??

Solo considera null y undefined como ausentes. Respeta 0, false y "" como valores válidos. La opción segura para defaults.

const n = config.port ?? 3000;
// Si port = 0, usa 0 (correcto!)

Optional Chaining ?.

Accede a propiedades profundas sin crashear si algún nivel es null o undefined. Esencial para datos de APIs que pueden ser incompletos.

const city = user?.address?.city;
// No lanza error si address es null

Asignación Lógica (ES2021)

Combina un operador lógico con asignación. Asigna solo si se cumple la condición — evita sobreescribir valores existentes.

config.timeout ||= 5000;  // Asigna si es falsy
config.debug &&= process.env.DEV;  // Asigna si es truthy
config.cache ??= new Map();  // Asigna si es null/undefined

Switch y Decisiones Múltiples

Switch tradicional

Útil para múltiples valores de una misma variable. Importante: sin break ocurre fall-through — el código sigue al siguiente case.

switch (estado) {
  case "activo": color = "green"; break;
  case "pausado": color = "yellow"; break;
  case "error": color = "red"; break;
  default: color = "gray";
}

Object Lookup (patrón moderno)

Reemplaza switch con un objeto donde cada clave es un caso. Más declarativo, inmutable, y fácil de extender sin tocar la lógica principal.

const colors = {
  activo: "green",
  pausado: "yellow",
  error: "red"
};
const color = colors[estado] ?? "gray";

Manejo de Errores: try / catch / finally

En producción, el código falla. Parsear JSON inválido, acceder a una API caída, dividir por cero — todo puede lanzar una excepción. El bloque try/catch permite reaccionar de forma controlada.

try {
  const data = JSON.parse(rawText);
  processUser(data);
} catch (err) {
  // err.name, err.message, err.stack
  console.error("Parsing failed:", err.message);
} finally {
  // Siempre se ejecuta: limpiar recursos
  loader.hide();
}
Tipos comunes de Error
  • SyntaxError — JSON mal formado, código inválido
  • TypeError — operación en tipo incorrecto
  • ReferenceError — variable no declarada
  • RangeError — valor fuera de rango válido
Errores personalizados
class ValidationError extends Error {
  constructor(msg) {
    super(msg);
    this.name = "ValidationError";
  }
}
throw new ValidationError("Email inválido");

Iteración Moderna: for...of vs for...in

El for clásico con índices numéricos es imperativo y verboso. ES6 introdujo alternativas más legibles y seguras.

BucleUsa conDevuelveCuándo usarlo
for...ofArrays, strings, Maps, SetsLos valores directamenteRecorrer colecciones iterables
for...inObjetosLas claves (keys)Inspeccionar propiedades de objetos
.forEach()Arraysundefined (sin break)Efectos secundarios sobre arrays
for clásicoCualquier cosaControl totalCuando necesitas el índice o break/continue
// for...of — itera valores
for (const producto of carrito) {
  console.log(producto.nombre, producto.precio);
}

// for...in — itera claves
for (const key in configObj) {
  if (configObj.hasOwnProperty(key)) console.log(key, configObj[key]);
}
