Condicional: if / else if / else

La estructura más fundamental del control de flujo. Evalúa una expresión booleana y ejecuta el bloque correspondiente. Puedes encadenar múltiples condiciones con else if.

Estructura completa

const nota = 75;

if (nota >= 90) {
  console.log("Excelente");
} else if (nota >= 60) {
  console.log("Aprobado");
} else {
  console.log("Reprobado");
}
Guard Clauses (cláusulas de guarda)

Retornan temprano cuando una condición falla. Evitan el anidamiento profundo (pyramid of doom).

// ❌ Anidado
function pagar(user) {
  if (user) {
    if (user.saldo > 0) {
      procesar();
    }
  }
}

// ✅ Guard clauses
function pagar(user) {
  if (!user) return;
  if (user.saldo <= 0) return;
  procesar();
}

Decisiones Compactas: Ternario y Cortocircuito

Ternario ? :

El if/else de una línea. Úsalo para asignaciones condicionales simples — no lo anides, pierde legibilidad.

const msg = edad >= 18
  ? "Pasa"
  : "Espera";

Operador OR ||

Retorna el primer valor truthy. Cuidado con 0 o cadenas vacías válidas — los trata como falsos.

const n = config.port || 3000;
// Si config.port = 0, usa 3000 (bug!)

Nullish ??

Solo considera null y undefined como ausentes. Respeta 0, false y "" como valores válidos.

const n = config.port ?? 3000;
// Si port = 0, usa 0 (correcto!)

Optional Chaining ?.

Accede a propiedades profundas sin crashear si algún nivel es null o undefined. Esencial para datos de APIs.

const city = user?.address?.city;
// No lanza error si address es null

Asignación Lógica (ES2021)

Combina operador lógico con asignación. Asigna solo si se cumple la condición.

config.timeout ||= 5000;  // si es falsy
config.debug &&= process.env.DEV;
config.cache ??= new Map();

Switch y Decisiones Múltiples

Switch tradicional

Compara una variable contra múltiples valores con igualdad estricta. Sin break ocurre fall-through.

switch (estado) {
  case "activo":
    color = "green";
    break;
  case "pausado":
    color = "yellow";
    break;
  case "error":
    color = "red";
    break;
  default:
    color = "gray";
}

Object Lookup (patrón moderno)

Reemplaza switch con un objeto donde cada clave es un caso. Más declarativo, inmutable, y fácil de extender.

const colors = {
  activo: "green",
  pausado: "yellow",
  error: "red"
};

const color = colors[estado] ?? "gray";
// ?? para el default, respeta false y 0

¿Cuándo usar switch vs if/else vs object lookup?

Manejo de Errores: try / catch / finally

En producción, el código falla. Parsear JSON inválido, acceder a una API caída, dividir por cero — todo puede lanzar una excepción. El bloque try/catch permite reaccionar de forma controlada.

try {
  const data = JSON.parse(rawText);
  processUser(data);
} catch (err) {
  // err.name, err.message, err.stack
  console.error("Parsing failed:", err.message);
} finally {
  // Siempre se ejecuta: limpiar recursos
  loader.hide();
}
Tipos comunes de Error
  • SyntaxError — JSON mal formado, código inválido
  • TypeError — operación en tipo incorrecto
  • ReferenceError — variable no declarada
  • RangeError — valor fuera de rango válido
Errores personalizados
class ValidationError extends Error {
  constructor(msg) {
    super(msg);
    this.name = "ValidationError";
  }
}
throw new ValidationError("Email inválido");

Bucles: de lo Clásico a lo Moderno

JavaScript tiene múltiples formas de iterar. Cada una sirve un propósito distinto. Conocer cuándo usar cada una es la diferencia entre código legible y código imperativo verboso.

Bucle for clásico

Tres partes: inicialización, condición y actualización. Control total sobre el índice — útil cuando necesitas el índice, iterar al revés, o saltar elementos con continue.

// Recorrer array con índice
for (let i = 0; i < alumnos.length; i++) {
  console.log(i, alumnos[i].nombre);
}

// Iterar al revés
for (let i = alumnos.length - 1; i >= 0; i--) {
  console.log(alumnos[i]);
}

// Saltar pares con continue
for (let i = 0; i < 10; i++) {
  if (i % 2 === 0) continue;
  console.log(i); // solo impares
}
Usa cuando
  • ✓ Necesitas el índice
  • ✓ Iterar al revés
  • ✓ Saltar elementos
  • ✓ Romper con break

Bucle while

Repite un bloque mientras la condición sea verdadera. Evalúa la condición antes de cada iteración — si es falsa desde el inicio, el cuerpo no se ejecuta. Ideal cuando no sabes de antemano cuántas vueltas necesitas.

// Reintentar conexión hasta lograrlo
let intentos = 0;
const MAX = 3;

while (intentos < MAX) {
  const ok = conectarServidor();
  if (ok) break;
  intentos++;
  console.log(`Reintento ${intentos}...`);
}

// Leer input hasta valor válido
let valor = null;
while (valor === null || valor < 0) {
  valor = obtenerInput();
}
Usa cuando
  • ✓ Iteraciones desconocidas
  • ✓ Esperar una condición
  • ✓ Reintentar operaciones
  • ⚠ Cuidar bucle infinito

Bucle do...while

Como while, pero evalúa la condición después del primer ciclo. Garantiza que el cuerpo se ejecute al menos una vez. Perfecto para validaciones de input donde siempre hay una primera solicitud.

// Pedir contraseña al menos una vez
let password;

do {
  password = prompt("Ingresa tu contraseña:");
} while (!esValida(password));

// Diferencia clave: siempre ejecuta una vez
let contador = 10;
do {
  console.log(contador); // imprime 10, aunque ya no cumpla
  contador++;
} while (contador < 10);
Usa cuando
  • ✓ Al menos 1 ejecución
  • ✓ Validar input de usuario
  • ✓ Menús interactivos
  • ✓ Inicialización + loop

Iteración Moderna: for...of, for...in y forEach

for...of — valores del iterable
for (const item of carrito) {
  console.log(item.nombre);
}

// También funciona con strings
for (const char of "hola") {
  console.log(char);
}
for...in — claves del objeto
const config = {
  host: "localhost",
  port: 3000
};

for (const key in config) {
  if (config.hasOwnProperty(key))
    console.log(key, config[key]);
}
forEach — callback por elemento
carrito.forEach((item, i) => {
  console.log(i, item.nombre);
  total += item.precio;
});

// No admite break/return para salir
