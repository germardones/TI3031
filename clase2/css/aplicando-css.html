Anatomía y Cascada CSS

Entiende la estructura de tus reglas y la jerarquía de poder en el navegador.

1. Anatomía de la Regla (El DNI del estilo)

Una regla CSS es una instrucción que le indica al navegador qué elementos debe modificar y cómo deben lucir. Pasa el mouse por cada parte para descubrir su rol.

h1Selector: ¿A quién quiero cambiar? {
  colorPropiedad: ¿Qué atributo voy a modificar?: blueValor: ¿Cómo quiero que luzca?;Punto y coma: El punto final de la instrucción
}

Resumen Técnico

  • Selector: Apunta al elemento HTML (Etiqueta, Clase o ID).
  • Bloque de Declaración: Todo lo que va entre llaves { }.
  • Declaración: La combinación de Propiedad + Valor.
  • Separador: Los : dividen la propiedad del valor.

2. Las 3 Vías de Implementación

Método Inline

"El Parche Directo"

Se escribe directamente dentro de la etiqueta HTML usando el atributo style.

<h1 style="color: red">...</h1>
¿Cuándo usar? Casos únicos o pruebas flash.
Método Internal

"Instrucciones Locales"

Se escribe dentro de la etiqueta <style> en el head del documento.

<style>
  h1 { color: blue; }
</style>
¿Cuándo usar? Landing pages de un solo archivo.
Método External

"El Manual Global"

Archivo .css independiente vinculado con la etiqueta <link>.

<link rel="stylesheet" href="style.css">
¿Cuándo usar? El estándar profesional para proyectos.
