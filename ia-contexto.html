Contexto y Prompts

Cómo estructurar una petición completa a una IA para obtener el resultado exacto que necesitas.

Anatomia de un Prompt Completo

Rol

Define quién es la IA. "Eres un desarrollador web senior..." Da personalidad y nivel de expertise.

Contexto

Proporciona el background relevante: "Estoy construyendo un formulario en HTML con validación..."

Tarea

La instrucción concreta. "Crea una función JavaScript que valide email y muestre errores inline."

Formato

Cómo quieres la respuesta: "Solo código, sin explicaciones. Usa ES6+. Comenta cada sección."

Ejemplo

Un ejemplo de entrada/salida esperada. "Input: 'hola@' → Output: Error: dominio inválido"

Principios de un Buen Contexto

  • Ser especifico: Cuanto mas detallado seas, mejor sera la respuesta. "Genera una funcion" es vago; "Genera una funcion JS que..." es preciso.
  • Una tarea a la vez: Los modelos funcionan mejor con instrucciones claras y focalizadas. Divide tareas complejas.
  • Dar ejemplos: Mostrar el formato deseado de entrada/salida reduce drasticamente la ambiguedad.
  • Iterar: El primer prompt raramente es el definitivo. Refina con "ahora hazlo mas corto" o "agrega manejo de errores".
  • Verificar siempre: Sin importar lo bien estructurado que este el prompt, valida el output antes de usarlo.
