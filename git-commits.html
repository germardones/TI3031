Conventional Commits

El estándar que hace tu historial legible para humanos y máquinas. Construye un commit perfecto paso a paso.

Constructor de Commits

Previsualización del commit
$ git commit -m " feat : tu descripción aquí "

Referencia de Tipos

Tipo Cuándo usarlo Ejemplo
feat:Nueva funcionalidad visible al usuariofeat(auth): agrega login con Google
fix:Corrección de un bugfix(navbar): corrige link roto en mobile
style:CSS/formato, sin cambio funcionalstyle: aplica dark mode al header
docs:Documentación únicamentedocs: actualiza instrucciones en README
refactor:Reestructura sin cambiar comportamientorefactor: extrae lógica de validación
test:Agrega o modifica pruebastest: agrega pruebas para formulario
chore:Mantenimiento del proyectochore: actualiza dependencias npm

Buenos vs Malos Commits

Commits a Evitar

"arregle cosas"
"WIP"
"no sé qué hice"

Commits Profesionales

feat(navbar): agrega menú hamburguesa
fix: corrige desborde en pantallas <375px
style(footer): actualiza tipografía
docs: agrega instrucciones de despliegue
chore: actualiza dependencias a v3.2
